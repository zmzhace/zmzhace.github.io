# 🚀 大语言模型(LLM)幻觉：分类、起因、检测与缓解机制

## 1. 幻觉分类

1. **事实幻觉 (Factual Hallucination)**
2. **忠实幻觉 (Faithfulness Hallucination)**

---

## 2. 幻觉起因

### 2.1 训练数据

#### 2.1.1 训练数据构成
- **预训练数据**
  - **有缺陷的预训练数据源中存在错误信息和偏差**
    - 模仿虚假 
    - 社会偏见
  - **预训练数据范围固有的知识边界**
    - 长尾分布 -> 私域领域知识缺失（可加入私域知识缓解）
    - 最新知识缺失 -> 联网检索解决
    - 版权敏感知识 -> 也属于知识缺失
- **对齐数据**
  - **劣质对齐数据引起的幻觉**
    - 部分研究表明，大模型难以从 SFT 过程中吸取新知识，且获取新知识与增加幻觉之间存在相关性。
    - 主要关注任务格式学习的特定任务指令往往会产生更高比例的幻觉反应。
    - 过于复杂和多样化的指令也会导致幻觉增加。

### 2.2 训练过程

#### 2.2.1 训练前产生的幻觉
- NTP (Next Token Prediction) 错误累积
- 注意力分数稀释

#### 2.2.2 监督微调 (SFT) 产生的幻觉
- SFT 寻求利用指令数据和相应的响应来解锁这些预先获得的能力。然而，当带注释的指令的需求超出模型预定义的能力边界时，就会出现挑战。
- 传统的 SFT 方法通常会强制模型完成每个响应，而不允许它们准确表达不确定性。

#### 2.2.3 RLHF 产生幻觉
- LLM 的激活封装了与其生成的陈述的真实性相关的内部信念。然而，这些内部信念和生成的输出之间偶尔会出现不一致。即使 LLM 根据人类反馈进行了改进，他们有时也会产生与其内部信念不同的输出。这种行为被称为**阿谀奉承 (Sycophancy)**，强调了该模型倾向于安抚人类评估者，而这往往是以牺牲真实性为代价的 -> 谄媚，预训练分布与 RLHF 分布不同。

### 2.3 推理过程

#### 2.3.1 不完善的解码策略
- 随机采样目前是这些 LLM 采用的主流解码策略。将随机性纳入解码策略的基本原理源于这样的认识：高似然序列通常会导致令人惊讶的低质量文本，这称为**似然陷阱**。解码策略的随机性带来的多样性是有代价的，因为它与幻觉风险的增加呈正相关。采样温度的升高会导致更均匀的令牌 (Token) 概率分布，从而增加从分布尾部以较低频率采样令牌的可能性。因此，这种对不经常出现的标记进行采样的倾向加剧了产生幻觉的风险。

#### 2.3.2 过度自信
- 该问题源于对部分生成的内容的过度关注，通常优先考虑流畅性，而牺牲了忠实于源上下文的代价。虽然主要采用因果语言模型架构的 LLM 已获得广泛使用，但过度自信的现象仍然持续存在。在生成过程中，下一个单词的预测取决于语言模型上下文和部分生成的文本。然而，正如先前的研究所证明的那样，语言模型通常在其注意机制中表现出局部焦点，优先考虑附近的单词，并导致上下文注意的显著缺陷。此外，这种担忧在 LLM 中进一步放大，因为他们倾向于生成冗长而全面的答复。在这种情况下，甚至更容易忘记指令的风险。这种注意力不足会直接导致忠实幻觉，其中模型输出的内容偏离了原始上下文。

#### 2.3.3 Softmax 瓶颈
- 大多数语言模型利用 softmax 层，该层对语言模型中最后一层的表示进行操作，并结合词嵌入来计算与词预测相关的最终概率。然而，基于 Softmax 的语言模型的功效受到了众所周知的 Softmax 瓶颈的限制，其中 Softmax 与分布式词嵌入的结合使用限制了给定上下文的输出概率分布的表达能力，从而阻止 LM 输出所需的分布。此外，Chang 和 McCallum 发现，当输出词嵌入空间内的期望分布表现出多种模式时，语言模型面临着准确地将所有模式中的单词优先排序为最上面的下一个单词的挑战，这也引入了幻觉的风险。

#### 2.3.4 推理失败
- 除了长尾知识的挑战之外，知识的有效利用与推理能力密不可分。例如，在多跳问答场景中，即使 LLM 拥有必要的知识，但如果问题之间存在多种关联，由于其推理的局限性，它可能很难产生准确的结果。此外，Berglund 等人揭示了 LLM 中一个被称为**逆转诅咒 (Reversal Curse)** 的特定推理失败。具体来说，虽然当问题被表述为“A 是 B”时，模型可以正确回答，但当被问到相反的“B 是 A”时，它会表现出失败的逻辑推论。这种推理上的差异超出了简单的推论。

## 3. 幻觉检测

### 3.1 事实幻觉检测

#### 3.1.1 涉及根据可信知识源验证生成的响应的真实性
- **事实提取**：涉及在模型输出中提取独立的事实陈述。
- **事实验证**：旨在根据可信知识源验证这些事实陈述的正确性。根据用于验证的知识源的类型，事实检查方法可以大致分为两个不同的部分：外部检索和内部检查。
  - **外部检索**
    - Min 等人开发了 FACTSCORE，这是一种专为评估长文本生成而定制的细粒度事实指标。它首先将生成内容分解为原子事实，然后计算可靠知识源支持的百分比。
    - Chern 等人扩展了这一概念。提出了一个统一的框架，使 LLM 能够利用一系列专门用于证据收集的外部工具来识别事实不准确之处。
    - 除了仅根据分解的主张检索支持证据外，Huo 等人通过查询扩展改进了检索过程。通过将原始问题与 LLM 生成的答案相结合，他们有效地解决了主题漂移问题，确保检索到的证据与问题和 LLM 的回答保持一致。 
  - **内部检查**
    - 鉴于其参数中编码的广泛事实知识，LLM 已被探索作为事实检查的事实知识来源。Dhuliawala 等人引入了验证链（CoVe），其中 LLM 首先为草稿响应生成验证问题，然后利用其参数知识来评估答案与原始响应的一致性，从而检测潜在的不一致之处。
    - Kadavath 等人和 Zhang 等人计算概率 p(True) 来评估布尔问题响应的真实性，完全依赖于模型的内部知识。
    - 此外，Li 等人观察到大多数原子语句是相互关联的，有些可能作为其他语句的上下文背景，这可能导致错误的判断。因此，他们指示 LLM 考虑所有事实陈述直接预测幻觉判断。
    - 然而，由于 LLM 本质上并不是可靠的事实数据库，仅依靠 LLM 的参数知识进行事实检查可能会导致评估不准确。 -> **预训练等训练和数据阶段就存在幻觉了，所以 LLM-as-a-Judge 可能不太靠谱。**

#### 3.1.2 不确定性估计，重点是通过内部不确定性信号检测事实不一致
- **LLM 内部状态**
  - LLM 的内部状态可以作为其不确定性的信息指标，通常通过令牌概率或熵等指标来体现。Varshney 等人通过考虑这些概念中的最小标记概率来确定模型对关键概念的不确定性。基本原理是，低概率是模型不确定性的有力指标，概念中存在的较高概率标记的影响较小。
  - 同样，Luo 等人采用了一种基于自我评估的方法来进行不确定性估计，其基本原理是语言模型能够从其生成的概念中熟练地重建原始概念，解释表明其对该概念的熟练程度。通过最初提示模型生成给定概念的解释，然后采用约束解码让模型根据其生成的解释重新创建原始概念，响应序列的概率得分可以充当该概念的熟悉度得分。
  - 此外，Yao 等人通过对抗性攻击的视角解释幻觉。利用基于梯度的令牌替换，他们设计了诱发幻觉的提示。值得注意的是，他们观察到，与来自对抗性攻击的令牌相比，由原始提示生成的第一个令牌通常表现出较低的熵。基于这一观察，他们建议设置一个熵阈值来定义此类幻觉攻击。 -> **PPL (Perplexity)**
- **LLM 行为**
  - 然而，当系统只能通过 API 调用访问时，可能无法访问输出的令牌级概率分布。考虑到这一限制，一些研究已将重点转移到通过自然语言提示或通过检查其行为表现来探索模型的不确定性。
  - 例如，Manakul 等人通过对 LLM 针对同一提示的多个回答进行抽样，通过评估事实陈述之间的一致性来检测幻觉。然而，这些方法主要依赖于直接查询，明确地从模型中获取信息或验证。
  - Agrawal 等人受到调查访谈的启发，主张使用间接询问。与直接问题不同，这些间接对应者经常提出开放式问题来引出具体信息。通过使用这些间接查询，可以更好地评估多个模型代之间的一致性。
  - 除了评估单个 LLM 多代的自我一致性的不确定性之外，还可以通过合并其他 LLM 来拥抱多主体视角。Cohen 等人从法律交叉询问实践中汲取灵感介绍了 LMvLM 方法。该策略利用审查者 LM 向考生 LM 提问，旨在揭示多轮交互过程中声明的不一致之处。

### 3.2 忠实幻觉检测

#### 3.2.1 基于事实
- 传统的基于 n-gram 的指标，例如 BLEU、ROUGE 和 PARENT-T，通常无法区分生成的内容和源内容之间的细微差异。
- 基于实体的度量通过计算实体的重叠进一步迈出了一步，因为这些关键实体的任何遗漏或不准确的生成都可能导致不忠实的响应。值得注意的是，即使实体匹配，它们之间的关系也可能是错误的。
- 因此，基于关系的度量关注关系元组的重叠，并引入一种度量来计算使用训练的端到端事实提取模型提取的关系元组的重叠。 -> **除了实体还需要关注关系以及语义。**

#### 3.2.2 基于分类器
- 基于分类器的指标。除了计算事实重叠之外，评估模型生成的真实性的另一种直接方法是利用根据相关任务（例如自然语言推理 NLI 和事实检查）的数据训练的分类器，或由综合特定任务的幻觉和真实内容组成的数据。
- 评估生成文本的忠实度的基本原则基于这样的想法：真正忠实的内容本质上应该由其源内容所蕴涵。与此相一致的是，大量研究在 NLI 数据集上训练分类器来识别事实不准确之处，特别是在抽象概括的背景下。
- 然而，Mishra 等人强调传统 NLI 数据集和不一致检测数据集之间输入粒度的不匹配限制了它们有效检测不一致的适用性。在此基础上，更先进的研究提出了一些方法，例如对对抗性数据集进行微调、在依赖弧级别分解蕴含决策、将文档分割成句子单元，然后聚合句子对之间的分数。
- 虽然使用相关任务的数据来微调分类器在评估可信度方面显示出了希望，但认识到相关任务与下游任务之间的固有差距至关重要。带注释数据的稀缺进一步限制了它们的适用性。为了应对这一挑战，大量研究探索利用数据增强方法来构建综合数据，通过基于规则的扰动或生成来微调分类器。 -> **用户 query 和模型生成之间用模型判断是否相关。**

#### 3.2.3 基于 QA
- 基于 QA 的指标。与基于分类器的指标相比，基于 QA 的指标最近因其捕获模型生成与其来源之间信息重叠的能力增强而受到关注。
- 这些指标的运作方式是首先从 LLM 输出的信息单元中选择目标答案，然后由问题生成模块生成问题。这些问题随后用于根据用户上下文生成源答案。最后，通过比较源答案和目标答案之间的匹配分数来计算 LLM 答案的忠实度。
- 尽管这些方法共享共同的主题方法，但它们在答案选择、问题生成和答案重叠等方面表现出可变性，从而导致不同的性能结果。在这项基础工作的基础上，Fabbri 等人对基于 QA 的指标中的各个组成部分进行了深入评估，进一步增强了可信度评估。

#### 3.2.4 基于不确定性
- 与用于检测事实幻觉的基于不确定性的方法相似，不确定性估计在评估可信度中的应用已被广泛探索，通常以熵和对数概率为特征。
- 对于基于熵的不确定性，Xiao 和 Wang 揭示了数据到文本生成中的幻觉可能性与预测不确定性之间的正相关性，这是由深度集成估计的。与此相关的是，Guerreiro 等人利用蒙特卡罗 Dropout 产生的假设方差作为神经机器翻译中的不确定性度量。最近，Vanderpoel 等人采用条件熵来评估抽象概括中的模型不确定性。
- 关于对数概率，它可以应用于不同的粒度级别，例如单词或句子级别。值得注意的是，一些研究采用了长度归一化序列对数概率来衡量模型置信度。
- 此外，考虑到当前面的上下文包含相同的幻觉信息时，幻觉标记可以被分配高概率，Zhang 等人重点关注信息最丰富、最重要的关键词，并引入惩罚机制来抵消幻觉内容的传播。

#### 3.2.5 基于 LLM
- 最近，LLM 卓越的指令跟踪能力突显了它们自动评估的潜力。利用这种能力，研究人员尝试了新的范式来评估模型生成内容的真实性。
- 通过为 LLM 提供具体的评估指南并向他们提供模型生成的内容和源内容，他们可以有效地评估忠诚度。最终的评估输出可以是忠诚度的二元判断，也可以是表示忠诚度的 k 点李克特量表。对于提示选择，评估提示可以是直接提示、思维链 (CoT) 提示、使用上下文学习或允许模型生成带有解释的评估结果。

## 5. 减轻与数据相关的幻觉

### 5.1 数据过滤
- 为了减少错误信息和偏见的存在，直观的方法包括从可靠来源仔细选择高质量的预训练数据。通过这种方式，我们可以确保数据的事实正确性，同时也最大限度地减少社会偏见的引入。早在 GPT-2 出现时，Radford 等人就提出了这一点，强调了专门抓取经过人类专家严格管理和过滤的网页的重要性。然而，随着预训练数据集的不断扩大，手动管理成为一个挑战。
- 鉴于学术或专业领域数据通常是准确的，收集高质量数据成为主要策略。值得注意的例子包括 Pile 和“教科书式”数据源。此外，事实证明，在预训练阶段对事实数据进行上采样可以有效提高 LLM 的事实正确性，从而减轻幻觉。
- 除了严格控制数据来源外，重复数据删除也是至关重要的环节。现有的实践通常分为两类：精确重复和近似重复。
  - 对于**精确重复**，最直接的方法涉及精确的子字符串匹配来识别相同的字符串。然而，考虑到预训练数据的巨大性，这个过程可能是计算密集型的，更有效的方法是利用后缀数组的构造，从而能够在线性时间内有效计算大量子串查询。
  - 对于**近似重复项**，识别通常涉及近似全文匹配，通常利用基于哈希的技术来识别具有显著 n 元语法重叠的文档对。此外，MinHash 作为大规模重复数据删除任务的流行算法脱颖而出。此外，SemDeDup 制作了使用预训练模型中的嵌入来识别语义重复，这是指具有语义相似但不相同的数据对。
- **讨论**：由于数据过滤直接针对幻觉的源头起作用，因此它可以通过确保使用高质量、事实上准确的来源来有效地减轻幻觉。尽管有效，但随着数据量的扩大，当前数据过滤方法的效率和可扩展性提出了重大挑战。此外，这些方法往往忽视了 LLM 生成内容的影响，这可能会带来新的风险和不准确性。为了取得进展，未来的研究必须集中于开发更高效、自动化的数据过滤算法，以跟上数据集的快速扩展和 LLM 生成内容的复杂性。

### 5.2 模型编辑
模型编辑引起了研究人员越来越多的关注，其目的是通过纳入额外的知识来纠正模型行为。当前的模型编辑技术可以分为两类：定位然后编辑和元学习。

- **定位然后编辑 (Locate-then-Edit)**
  - 定位然后编辑方法由两个阶段组成，首先定位模型参数的“有缺陷”部分，然后对其应用更新以改变模型的行为。
  - 例如，ROME 通过破坏并随后恢复激活来定位编辑相关层，然后以直接方式更新 FFN 的参数来编辑知识。 
  - MEMIT 采用与 ROME 相同的知识定位方法，能够实现多层的并发更新，以方便数千个编辑知识的同时集成。然而，Yao 等人发现这些方法缺乏重要的泛化能力以及对不同模型架构的不同性能和适用性。根据经验，性能最佳的方法 ROME 和 MEMIT 仅适用于仅解码器的 LLM。
- **元学习 (Meta-learning)**
  - 元学习方法训练外部超网络来预测原始模型的权重更新。然而，元学习方法通常需要额外的训练和内存成本。
  - MEND 利用低秩分解和专门的设计来减少超网络的大小。值得注意的是，MEND 会表现出抵消效应，其中对应于不同键的参数变化会显著地相互抵消。
  - MALMEN 进一步解决了这个问题，将参数移位聚合构建为最小二乘问题而不是简单的求和，从而大大增强了其广泛编辑的能力。虽然这些方法可以细粒度地调整模型的行为，但对参数的修改可能会对模型的固有知识产生潜在的有害影响。
- **讨论**：模型编辑提供了一种精确的方法来减轻由特定错误信息引起的幻觉，而无需进行大量的再训练。然而，这些方法难以应对大规模更新，并且可能会对模型的整体性能产生不利影响，特别是在应用连续编辑时。因此，未来的研究应侧重于改进模型编辑，以更有效地处理大规模知识更新并解决社会偏见引起的幻觉。

### 5.3 检索生成增强 (RAG)

#### 5.3.1 检索增强生成
- 通常，检索增强生成（RAG）遵循**检索然后读取 (Retrieve-then-Read)** 的管道，其中相关知识首先由检索器从外部源检索，然后由生成器根据用户查询和检索到的文档生成最终响应。通过将外部知识与 LLM 解耦，RAG 可以有效缓解知识鸿沟带来的幻觉，同时又不影响 LLM 的表现。常见的做法根据检索的时机可以分为三部分：一次性检索、迭代检索和事后检索。

#### 5.3.2 一次性检索
- 一次性检索旨在将单次检索获得的外部知识直接添加到 LLM 的提示中。Ram 等人引入了上下文中的 RALM，它需要一种简单而有效的策略，将所选文档添加到 LLM 的输入文本中。
- 除了维基百科等传统知识存储库之外，正在进行的研究工作还探索了替代途径，特别是知识图谱 (KG)。这些知识图谱是促进 LLM 的关键工具，促进他们与最新知识的互动，并引出强大的推理路径。
- Varshney 等人引入参数化知识指导（PKG）框架，利用特定领域的知识增强 LLM。PKG 采用可训练的背景知识模块，将其与任务知识相结合并生成相关的上下文信息。  

#### 5.3.3 迭代检索
- 当面临多步推理和长篇问答等复杂挑战时，传统的一次性检索可能会出现不足。为了满足这些苛刻的信息需求，最近的研究提出了迭代检索，它允许在整个生成过程中不断收集知识。
- 认识到思想链 (CoT) 提示给 LLM 在多步骤推理方面带来的实质性进步，许多研究尝试在每个推理步骤中融入外部知识，并进一步指导基于持续推理的检索过程，减少推理链中的事实错误。Press 等人以思维链提示为基础介绍自我询问。与传统的连续的、未界定的思维链提示不同，自我询问会在每一步中勾勒出它打算解决的问题，随后根据后续问题进行搜索操作。
- Feng 等人并没有仅仅依靠思想链提示来进行检索指导，Shao 等人采用了迭代检索生成协作框架，其中模型的响应作为有洞察力的上下文来获取更多相关知识，随后在后续迭代中完善响应。
- 除了多步推理任务之外，Jiang 等人将他们的重点转移到长格式生成上。他们提出了一个主动检索增强生成框架，该框架迭代地将即将到来的预测视为检索相关文档的查询。如果预测包含低置信度的标记，则该句子将进行重新生成。
- 除了使用迭代检索来改进中间代之外，Zhang 等人提出了 MixAlign，它使用基于模型的指导迭代地细化用户问题并寻求用户的澄清，最终增强问题和知识之间的一致性。

#### 5.3.4 事后检索
- 除了传统的检索然后阅读范式之外，一系列工作还深入研究了事后检索，通过后续基于检索的修订来完善 LLM 输出。 
- 为了增强 LLM 的可信度和归属，Gao 等人采用了研究然后修改的工作流程，首先研究相关证据，然后根据发现的证据差异修改初始生成。
- 同样，Zhao 等人引入了验证和编辑框架，通过结合外部知识来增强推理链的事实准确性。对于一致性低于平均水平的推理链，该框架会生成验证问题，然后根据检索到的知识提炼基本原理，确保给出更真实的回答。
- Yu 等人通过多样化的答案生成增强了事后检索方法。他们不是只生成一个答案，而是对各种潜在答案进行采样，从而获得更全面的检索反馈。此外，通过采用考虑检索前后答案可能性的集成技术，它们进一步降低了误导性检索反馈的风险。

## 6. 相关路径拓展思考 (Related Path)

### 6.1 逻辑一致性与 CoT
- 受人类思维过程的启发，引入了思想链 (CoT) 来鼓励 LLM 将复杂问题分解为明确的中间步骤，从而提高推理过程的可靠性。尽管有效，但最近的研究表明，LLM 产生的中间原理并没有忠实地捕捉其潜在行为。一个研究分支受到启发，旨在提高 LLM 生成的中间原理的一致性，特别是在多步骤推理和逻辑推理中。
- 为了增强思想链的自我一致性，Wang 等人采用了知识蒸馏框架。他们首先使用对比解码生成一致的基本原理，然后使用反事实推理目标微调学生模型，这有效地消除了在不考虑基本原理的情况下得出答案的推理捷径。此外，通过直接采用对比解码，LLM 可以减少表面级复制并防止错过推理步骤。
- 此外，Li 等人对不忠实推理过程中的上下文、CoT 和答案之间的因果相关性进行了深入分析。分析发现，不忠问题在于 CoT 获取的上下文信息与答案不一致。为了解决这个问题，他们提出了**推理桥接 (Reasoning Bridging)**，以归因方法来回忆上下文信息作为提示，以增强 CoT 推理并过滤掉语义一致性和上下文归因分数较低的嘈杂 CoT。
- 直接偏好优化 (DPO) 使 LLM 倾向于选择正确的推理链而不是反事实链，并且推理模块鼓励 LLM 使用反事实和因果偏好目标忠实地推理推理步骤。
- 与自然语言推理相比，逻辑推理需要严格的逻辑计算，而纯文本往往缺乏精确的逻辑结构，导致推理不忠实。为了解决这个问题，Xu 等人引入了**符号 CoT (SymbCoT)**，它将符号表达式合并到 CoT 中来描述中间推理步骤。具体来说，SymbCoT 将自然语言上下文翻译成符号表示，然后制定分步计划来解决逻辑推理问题，然后由验证者检查翻译和推理链，从而确保忠实的逻辑推理。
- **讨论**：忠实增强解码显著提高了 LLM 输出与所提供上下文的一致性，并增强了生成内容的内部一致性。然而，上下文感知解码等策略通常缺乏自适应机制，限制了它们在需要动态关注上下文的场景中的有效性。此外，许多解码策略需要集成不关注上下文的附加模型，从而引入显著的计算开销并降低效率。

### 6.2 RAG 缓解幻觉的局限性
- 检索增强生成（RAG）已成为一种有前途的策略，可以减轻幻觉并提高 LLM 输出的真实性。通过在推理过程中纳入大规模的外部知识库，RAG 为 LLM 配备了最新的知识，从而降低了 LLM 因固有知识边界而产生幻觉的潜在风险。
- 尽管旨在减轻 LLM 的幻觉，检索增强 LLM 仍然会产生幻觉。RAG 中的幻觉表现出相当大的复杂性，表现为事实上不准确或具有误导性的输出。当 LLM 生成的内容与现实世界的事实不符、无法准确反映用户的查询或不受检索到的信息支持时，就会出现这些幻觉。
- 这种幻觉可能源于两个主要因素：**检索失败**和**生成瓶颈**。通过对当前 RAG 系统存在的局限性进行全面分析，我们的目标是揭示检索增强 LLM 的潜在改进，为更可靠的信息检索系统铺平道路。
