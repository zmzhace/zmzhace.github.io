# 大语言模型(LLM)幻觉检测：白盒、灰盒与黑盒方案及评估反思

## 一、 事实幻觉检测 (Factual Hallucination Detection)

**核心目标**：评估模型生成的内容是否符合客观世界（或企业内部）的真实情况。这类方法通常从模型的不确定性或内部特征中寻找“说谎/心虚”的信号。

### 1. 白盒检测 (White-box: 深入模型“大脑”与内部状态)

*前提：需要完全访问模型的权重、隐藏状态或注意力机制。*

- **单层/局部隐藏状态分析**：提取特定层（通常是最后一层）的 Token 内部状态特征。代表方法：HADEMIF 将隐藏状态输入 MLP 自动学习幻觉映射；HDM-2 发现中间层（如第 25 层）比输出层更能有效区分“通用知识”真伪。
    
- **细粒度非线性探测与层搜索 (Neural Probes)**：传统线性探针只能捕捉浅层语义。前沿方法在冻结大模型参数时，插入多层感知机 (MLP) 探针进行非线性建模，实现 Token（实体）级别的实时幻觉定位。同时，利用贝叶斯优化算法自动搜索最佳的探针插入层，取代人工经验盲猜。
    
- **跨层动态轨迹建模 (Dynamic Trajectories)**：不孤立看单个 Token，而是将整句话的生成视为连续的特征流动。代表方法：HD-NDEs 利用神经常微分方程拟合降维后的隐藏状态轨迹，精准捕获潜伏在句子中段的幻觉。
    
- **注意力机制与图论 (Attention Maps & Graph Theory)**：将信息流抽象为网络结构。代表方法：LapEigvals 将注意力矩阵转化为图的拉普拉斯矩阵，提取特征值，发现幻觉产生时往往伴随信息流动的“瓶颈”或结构异常。
    
- **表示空间维度分析**：代表方法：eRank 发现当隐藏层表示空间的维度“坍缩”时，预示着模型正在脱离上下文产生幻觉。

### 2. 灰盒检测 (Grey-box: 依赖输出层的概率分布)

*前提：需要通过 API 或模型获取输出 Token 的概率、Logits 等分布数据。*

- **预测特征融合**：代表方法：HADEMIF 提取 Token 概率（最大/最小/均值）、预测裕度 (Margin)、Logits 范数等，输入 D3T（深度动态决策树）进行可解释分类。
    
- **基于熵的不确定性估计**：计算预测熵、长度归一化熵或语义熵，熵值越高代表模型越“犹豫”。
    
- **共形预测 (Conformal Prediction)**：代表方法：KnowNo 框架，提供统计保证，当概率阈值表明模型对决策极度不确定时，触发人工介入。

### 3. 黑盒检测 (Black-box: 仅依赖文本输入与输出)

*前提：只能通过常规对话 API 获取最终的文本回复。*

- **多维评估融合的独立判别器 (Independent Discriminators)**：为克服单一指标（如 ROUGE）的误报，专门训练轻量级预训练模型（如 RelD 框架）作为综合裁判。在工程实现上，直接用回归拟合人类打分效果极差，通常将其转化为多分类任务（如十分类），再通过加权平均概率转回二元判别，能大幅提升检测的鲁棒性。
    
- **自我一致性交叉验证**：对同一 Prompt 多次采样，或利用多智能体相互盘问，寻找自相矛盾点 (如 SelfCheckGPT)。
    
- **对抗性提示 (Adversarial Prompting)**：故意修改提示词诱导模型犯错，测试其事实鲁棒性。
    
- **逻辑与约束求解器验证**：将文本转化为规则，输入外部求解器（如 SMT）检查是否违反物理常识或硬性约束。
    
- **极简基准：基于长度的启发式检测 (Length-based Baselines)**：代表发现：幻觉回答通常具有“更长且长度波动更大”的显著特征（雪球效应）。仅凭单次长度或多次生成长度的标准差，就能达到极具竞争力的检测效果。

## 二、 忠实幻觉检测 (Faithful / Contextual Hallucination Detection)

**核心目标**：评估模型是否忠实于给定的上下文（如 RAG 检索出的文档、长文本背景），防止“无中生有”或“自相矛盾”。

### 1. 细粒度多任务分类器 (Task-Specific Classifiers)

- **企业级分阶段验证架构**：代表方法：HDM-2 提出全新分类法。先验证是否忠实于检索文档；若不忠实，再降级验证是否属于正确的“企业专有知识”或“通用知识”；同时过滤掉“无害闲聊”。
    
- **针对长文本的“分块-聚合”架构**：解决长上下文幻觉。将超长文本切块，由小型编码器分别提取表征，再通过全局注意力机制对比生成的响应，精准定位“矛盾”或“无根据”的信息。

### 2. 大语言模型作为裁判 (LLM-as-a-Judge)

- **基础判别机制**：在 EMNLP 2025 的大规模评测中被证实是目前唯一稳定、可靠的评判标准。放弃不可靠的 N-gram 重叠度，直接利用 GPT-4 或类似先进模型，给予明确 Prompt 让其判断是否脱离上下文。
    
- **具备可解释性的深度裁判 (Explainable Judge)**：不仅输出“真/假”，还能输出归因解释。代表方法：HuDEx 框架通过 LoRA 微调轻量级模型，配合“幻觉检测专家”的设定和结构化的多阶段任务提示词（审查事实 -> 比对逻辑 -> 输出判定与解释），让裁判能用自然语言详细解释错因，形成有助于模型优化的负反馈闭环。

### 3. 基于问答框架 (QA-based)

- **机制**：逆向提问法（如 $Q^2$）。从生成内容中出题，在源文档中找答案，对比两者是否匹配。

## 三、 幻觉检测的新兴理论与评估反思 (Emerging Theories & Meta-Evaluation)

这批新论文对行业的底层逻辑进行了极好的梳理，可以总结为三大核心原则：

- **理论极限原则：负样本是必不可少的**。耶鲁大学研究从计算学习理论证明：如果只给模型喂正确的数据，自动化幻觉检测在理论上是不可能的。必须引入专家标注的错误/负向反馈，幻觉检测才具有可行性。
    
- **指标祛魅原则：警惕复杂的“纸面指标”**。研究共同指出：很多复杂的检测算法实际上只是在测算“回复长度”；传统的重叠度指标极度不可靠。建议不要过度追求复杂的检测工程；模式寻求型解码比随机采样产生幻觉的概率更低。
    
- **场景定制原则：从问答向“决策系统”延伸**。自动驾驶和机器人等 Agent 系统的幻觉评估，必须超越简单的“真假”，引入四个新维度：合规性 (物理约束)、期望性 (最高效率)、相关性与合理性。
